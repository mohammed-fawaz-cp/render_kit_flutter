# RenderKit CLI Specification & User Guide

The RenderKit CLI (`renderkit_cli`) is the primary developer toolchain utility for managing environments, configurations, builds, and preview modes in RenderKit projects.

---

## 🚀 Running the CLI

If you have added `renderkit_cli` to your package's `dev_dependencies` via a local path:
```yaml
dev_dependencies:
  renderkit_cli:
    path: ../renderkit_cli
```

You can execute the CLI by running:
```bash
dart run renderkit_cli <command> [arguments]
```

---

## 🛠️ Commands Reference

### 1. `configure`
Setup the compiler configuration file (`build.yaml`) for your project.

* **Purpose:** Ensures the Dart `build_runner` framework knows to trigger the RenderKit code generator builder on your source files.
* **Functionality:** Writes a default `build.yaml` target configuration if one does not exist.
* **Example Output:**
  ```bash
  $ dart run renderkit_cli configure
  ⚙️ Configuring RenderKit...
  ✅ Created build.yaml config file.
  ```

---

### 2. `install`
Installs required compiler toolchain configurations.

* **Purpose:** Validates the dependencies required for target native compilation and ensures `renderkit` & `renderkit_generator` are present.
* **Example Output:**
  ```bash
  $ dart run renderkit_cli install
  📦 Installing dependencies...
  ✅ RenderKit core DSL dependency added.
  ✅ renderkit_generator dev_dependency added.
  ```

---

### 3. `doctor`
Runs native environment diagnostics for Android (Jetpack Compose) and iOS (SwiftUI).

* **Purpose:** Inspects native project subfolders to prevent compilation or runtime errors during native app builds.
* **Checks Performed:**
  * **Android:** Verifies that Jetpack Compose is enabled and Material3 dependencies are declared in the Android app Gradle configuration file (`android/app/build.gradle` or `android/app/build.gradle.kts`).
  
    #### Manual Android Gradle Configuration
    
    ##### Groovy DSL (`android/app/build.gradle`):
    ```groovy
    android {
        buildFeatures {
            compose true
        }
        composeOptions {
            kotlinCompilerExtensionVersion '1.5.0'
        }
    }
    dependencies {
        implementation 'androidx.compose.material3:material3:1.1.0'
    }
    ```
    
    ##### Kotlin DSL (`android/app/build.gradle.kts`):
    ```kotlin
    android {
        buildFeatures {
            compose = true
        }
        composeOptions {
            kotlinCompilerExtensionVersion = "1.5.0"
        }
    }
    dependencies {
        implementation("androidx.compose.material3:material3:1.1.0")
    }
    ```
    
  * **iOS:** Confirms if the iOS deployment target is at least version `13.0` inside `ios/Podfile` (SwiftUI requires iOS 13+).
* **Example Output:**
  ```bash
  $ dart run renderkit_cli doctor
  🏥 Running RenderKit Diagnostics...
  🤖 [Android Environment]
    ✅ Jetpack Compose is enabled.
    ✅ Material3 Compose dependency found.
  🍎 [iOS Environment]
    ✅ iOS Deployment Target is 13.0 (>= 13.0).

  🎉 All environment checks passed! RenderKit is ready to build.
  ```

---

### 4. `generate`
Manually compiles the Dart DSL code to native files.

* **Purpose:** Triggers the code generator builder which parses all `@RenderEntry()` widgets and writes native `.compose.kt` (Kotlin) and `.swift` (Swift) files.
* **Example Output:**
  ```bash
  $ dart run renderkit_cli generate
  🚀 Starting RenderKit compilation...
  [INFO] Generating SDK layouts...
  ✅ Compilation completed.
  ```

---

### 5. `preview`
Spins up the hot-reloadable Flutter preview renderer.

* **Purpose:** Allows previewing compiled UI designs interactively within a Flutter application without needing to load or compile the native Android or iOS app code on every edit.
* **Example Output:**
  ```bash
  $ dart run renderkit_cli preview
  📱 Spinning up Flutter Preview server...
  ℹ️ Run your flutter app and wrap your widget with RenderPreview.
  ```

---

### 6. `validate`
Validates the current widget configurations and bindings.

* **Purpose:** Performs compilation checks to ensure there are no duplicate actions or missing/unresolved state bindings.
* **Example Output:**
  ```bash
  $ dart run renderkit_cli validate
  🔍 Validating RenderKit widget schemas...
  ✅ No duplicate actions found.
  ✅ All state bindings are resolved.
  ```

---

### 7. `clean`
Cleans up the compiler cache and build assets.

* **Purpose:** Clears the build cache generated by `build_runner` to resolve caching conflicts or force a clean compilation run.
* **Example Output:**
  ```bash
  $ dart run renderkit_cli clean
  🧹 Cleaning build runner cache...
  ✅ Done.
  ```
